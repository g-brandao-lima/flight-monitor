## Integration Issues

### Issue 1 (Critical): func.strftime SQLite-only in dashboard_service.py

**Lines:** 91, 151
**What:** `func.strftime("%w", ...)` e `func.strftime("%Y-%m-%d %H", ...)` sao funcoes SQL do SQLite. PostgreSQL nao tem strftime.
**Impact:** Dashboard index (`GET /`) crasha com `ProgrammingError` em producao (PostgreSQL) para qualquer usuario com dados de snapshot. Invisivel nos testes (rodam em SQLite).
**Fix:** Substituir por `func.extract("dow", ...)` e expressao compativel com PostgreSQL, ou usar funcoes SQLAlchemy dialect-agnostic.
**Affected:** DB-01, MULTI-01, MULTI-03

### Issue 2 (Medium): APP_BASE_URL ausente do render.yaml

**What:** `Settings.app_base_url` tem default `http://localhost:8000`. Nao esta declarado como env var no render.yaml.
**Impact:** Links de silenciar alerta nos emails apontam para localhost em producao. Email chega no destinatario correto (MULTI-02 parcial), mas o link e inacessivel.
**Fix:** Adicionar `APP_BASE_URL` com `sync: false` ao render.yaml.
**Affected:** MULTI-02
